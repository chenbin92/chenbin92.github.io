# 视图

React 核心特点之一就是通过组件化构成复杂的 UI 界面，如下图所示页面均是由不同的组件组合而成，这很好的解决了页面之间的复用和对一个大型应用进行划分。

在实际开发过程中，除了组件之外构成视图的还有 HTML 模板、布局和页面等内容，ice.js 进一步对这些进行了约定和规范。

# HTML 模板

你可以定制化 ice.js 默认的应用模板。

定制化默认的 html 模板，只需要在应用的 public 目录下创建一个 index.html 的文件。

默认模板为：

<!DOCTYPE html>
<html {{ HTML_ATTRS }}>
  <head {{ HEAD_ATTRS }}>
    {{ HEAD }}
  </head>
  <body {{ BODY_ATTRS }}>
    {{ APP }}
  </body>
</html>

举个例子，你可以修改模板添加 IE 的条件表达式：

<!DOCTYPE html>
<!--[if IE 9]><html lang="en-US" class="lt-ie9 ie9" {{ HTML_ATTRS }}><![endif]-->
<!--[if (gt IE 9)|!(IE)]><!--><html {{ HTML_ATTRS }}><!--<![endif]-->
  <head {{ HEAD_ATTRS }}>
    {{ HEAD }}
  </head>
  <body {{ BODY_ATTRS }}>
    {{ APP }}
  </body>
</html>

# 布局

ice.js 允许你扩展默认的布局，或在 layout 目录下创建自定义的布局。并且 layouts 根目录下的所有文件都属于个性化布局文件，可以在页面组件中利用 layout 属性来引用。

如在视图中通过 app 实例获取 layout 对象：

export default = () => {
  const { app } = useApp()
  
  // 返回 layouts 目录下的布局名称
  console.log(app.layouts)
}

# 自定义布局

layouts 目录中的每个文件 (顶级) 都将创建一个可通过页面组件中的 layout 属性访问的自定义布局。

# 页面

页面实际上也是组件，只不过页面常常是组件的容器以及与路由和数据进行关联。ice.js 为页面提供了一些特殊的配置项，以便你能快速开发的开发应用。

export default () => {

  // 配置当前页面的 Meta 标签
  head () {
    title: "",
    meta: [
      {
        hid: 'description',
        name: 'description',
        content: 'My custom description'
      }
    ]
  },

  // 约定式路由下指定当前页面使用的布局（layouts 根目录下的布局文件）
  layout () {
    return 'LoginLayout'
  }

  return (
    <>
      {/* you jsx here... */}
    </>
  )
}
