# 状态管理

Nice.js 内置集成了 icestore 状态管理方案，并在此基础上进一步遵循 “约定优于配置” 原则，进行抽象和封装，使得开发变得非常容易，就像用程序写 Hello World 一样简单。

# 状态管理约定

框架约定了状态主要分为全局状态和页面状态，目录组织如下：

├── src/
│   ├── pages/ 
│   │   └── Home/                  # 页面单元
│   │       ├── index.tsx 
│   │       ├── service.ts
│   │       ├── store.ts           # 页面状态
│   │       └── index.module.scss 
│   ├── store/                     # 全局状态
│   │   ├── storeA.js          
│   │   └── storeB.js

# 页面状态

页面状态顾名思义以页面为维度进程拆分，数据模型默认需要导出一个函数，框架会自动传入 App 实例，使用方式如下：

// pageA/store.js

export default (app) => {
  const { service } = app;

  return {
    // 默认以 page 文件名称作为 namespace
    namespace: 'pageA'

    // 数据对象
    data: {},

    // 异步请求
    async getUserInfo() {
      this.data = await service.getUserInfo()
    }
  }
}

# 全局状态

全局状态即为多个页面之间共享的数据，框架约定全局共享的数据放在 src/store 目录下，数据模型默认需要导出一个函数，框架会自动传入 App 实例，使用方式如下：

// src/store/index.js

export default (app) => {
  const { service } = app;

  return {
    // 默认以 global 文件名称作为 namespace
    namespace: 'global'

    // 数据对象
    data: {},

    // 异步请求
    async getUserInfo() {
      this.data = await service.login()
    }
  }
}
