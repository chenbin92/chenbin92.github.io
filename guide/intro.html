# ice.js 是什么

ice.js 是一个基于 React.js 的通用框架。

在 React v16.8 中加入 hooks 这一重大改进之后，React 之后版本的多数更改都相对较小。到了 2019 年末，版本更新了 16.14， hooks 的重大改进发布之后，React 团队开始转移重心，希望提供更多工具来改善开发体验。实际上，开发体验正式 React Conf 2019 的主题。 React Conf 的主讲人，React 团队开发经理 Tom Occhion 表示，开发体验主要围绕以下三件事：准入门槛低，生产率高和扩展能力强。

随着 React 生态的快速发展，基于 React 的方案多种多样，据不完全统计在 Hook 发布之后，状态管理方案就有 100 多个，这意味着很多方案开发者依然要做很多选择，没有约定的团队，沟通成本和跨团队协作成本是非常高的，这时候统一一套规范能力就显得极为需要。

而 ice.js 正是在这种背景下诞生的，通过框架提供完整的标准化的 React 应用开发模式和最佳实践。目的是简化应用的开发，以及收敛技术栈、屏蔽底层差异和统一开发体验。

# 设计原则

在框架设计上，ice.js 借鉴了 Rails 的设计理念

  • 不要自我重复（DRY）： DRY 是软件开发中的一个原则，意思是“系统中的每个功能都要具有单一、准确、可信的实现。”。不重复表述同一件事，写出的代码才更易维护、更具扩展性，也更不容易出问题。
  • 约定优于配置： ice.js 为 Web 应用的大多数需求都提供了最好的解决方法，并且默认使用这些约定，而不是在长长的配置文件中设置每个细节。

# 特性

使用 ice.js 我们可以按照框架的最佳实践规范和约定，非常快速的开始 React 项目的开发，通过配置应用类型将源代码分别编译出 SPA、MPA、SSR、Serverless、微前端等不同的应用类型。
