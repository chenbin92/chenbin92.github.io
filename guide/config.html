# 配置

框架提供了强大且可扩展的配置功能，可以自动合并应用、插件、框架的配置，按顺序覆盖，且可以根据环境维护不同的配置。合并后的配置可直接从 app.config 获取。

# 配置文件

框架约定配置文件目录格式如下：

src
├── config
│   ├── default.ts
│   ├── prod.ts
│   └── dev.ts

在 App 实例化会自动传入和合并这些配置项：

const app = new App({
  config
})

# 配置写法

配置文件返回的是一个 object 对象，可以覆盖框架的一些配置，应用也可以将自己业务的配置放到这里方便管理。

export default {
  APP_ID: ''
}

# 获取配置

框架会合并配置并将配置挂载到 App 实例上，因此只要能获取到 App 实例的位置就能获取到 config 对象。

  • 在视图中获取配置
export default = () => {
  const app = useApp()
  
  const { config } = app;

  // console.log(config);
}
  • 在模型中获取配置
export default = (app) => {  
  const { config } = app;

  // console.log(config);
}
