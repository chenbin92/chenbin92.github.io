# 样式方案

# 全局样式

对于整个项目的全局样式，统一定义在 src/global.scss 文件中，框架会默认引入改文件：

// 引入默认全局样式
@import '@alifd/next/reset.scss';

body {
  background: #000;
}

# 局部样式

在页面或者组件中我们推荐使用 sass + CSS Modules 的方式添加样式，可以有效解决样式冲突等问题，书写方式如下：

首先在组件目录下新建 index.jsx 和 index.module.scss 两个文件：

// ./components/Card/index.module.scss
.container {
  background: #fff;
}

.title {
  font-size: 12px;
}

在 jsx 文件中引入对应的样式文件，并将 className 与对应样式关联：

// ./components/Card/index.jsx
import styles from './index.module.scss';

function Card() {
  return (
    <div className={styles.container}>
      <h2 className={styles.title}>CSS Modules</h2>
    </div>
  );
}

使用该方案之后，上文中的 nick、userCard 这种 className 都会被编译为唯一性的名字，避免因为重名 className 而产生样式冲突。

工程上也支持 less/stylus 等方案，开发者可按需使用。
