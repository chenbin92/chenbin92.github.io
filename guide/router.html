# 路由

Nice.js 依据 pages 目录结构自动生成路由配置，也即约定式路由。

# 基础路由

假设 pages 的目录结构如下：

pages/
--| user/
-----| index.js
-----| one.js
--| index.js

那么，Nice.js 自动生成的路由配置如下：

export default [
  {
    name: 'index',
    path: '/',
    component: 'pages/index'
  },
  {
    name: 'user',
    path: '/user',
    component: 'pages/user/index'
  },
  {
    name: 'user-one',
    path: '/user/one',
    component: 'pages/user/one'
  }
]

# 动态路由

在 Nice.js 里面定义带参数的动态路由，需要创建对应的以下划线作为前缀的 JSX 文件 或 目录。

pages/
--| _slug/
-----| comments.jsx
-----| index.jsx
--| users/
-----| _id.jsx
--| index.jsx

Nice.js 生成对应的路由配置表为：

export default [
  {
    name: 'index',
    path: '/',
    component: 'pages/index'
  },
  {
    name: 'users-id',
    path: '/users/:id?',
    component: 'pages/users/_id'
  },
  {
    name: 'slug',
    path: '/:slug',
    component: 'pages/_slug/index'
  },
  {
    name: 'slug-comments',
    path: '/:slug/comments',
    component: 'pages/_slug/comments'
  }
]

你会发现名称为 users-id 的路由路径带有 :id? 参数，表示该路由是可选的。如果你想将它设置为必选的路由，需要在 users/_id 目录内创建一个 index.js 文件。

# 嵌套路由

创建内嵌子路由，你需要添加一个 JSX 文件，同时添加一个与该文件同名的目录用来存放子视图组件。

假设文件结构如：

pages/
--| users/
-----| _id.js
-----| index.js
--| users.js

Nice.js 自动生成的路由配置如下：

export default [
  {
    path: '/users',
    component: 'pages/users',
    children: [
      {
        path: '',
        component: 'pages/users/index',
        name: 'users'
      },
      {
        path: ':id',
        component: 'pages/users/_id',
        name: 'users-id'
      }
    ]
  }
]
