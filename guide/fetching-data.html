# 数据请求

大多数前端应用都需要通过 HTTP 协议与后端服务器通讯。框架内置提供了 fetch 功能，它基于社区主流的 useSWR 和 axios 进行封装，提供了数据请求功能。

# fetch 方法

fetch 方法用于数据请求，默认挂载在 app 示例对象下，使用方式如下：

export default = (app) => {
  const { fetch, logger } = app

  fetch({
    url: '/api/getUserInfo',
    method: 'GET'
  }).then(res => {
    logger.success(res)
  }).catch(err => {
    logger.error(err)
  })
}

# 数据请求约定

框架约定了数据请求的方式，以及全局请求和页面请求的接口定义，目录组织如下：

├── src/
│   ├── pages/ 
│   │   └── Home/                  # 页面单元
│   │       ├── index.tsx          # 入口
│   │       ├── service.ts         # 服务
│   │       ├── store.ts           # 模型
│   │       └── index.module.scss 
│   ├── service/                   # 全局服务接口        
│   │   ├── serviceA.js          
│   │   └── serviceB.js
│   ├── store/                     # 全局模型定义
│   │   ├── storeA.js          
│   │   └── storeB.js

# 请求接口定义

接口定义规范如下，默认需要导出一个函数，该函数接收全局的 App 实例。

// pageA/service.js

export default (app) => {
  // 从 app 实例上获取 fetch 方法
  const { fetch } = app;

  return {
    getUserInfo() {
      return app.fetch('/api/userInfo')
    }
  }
}

# 在模型中调用接口服务

通常数据请求的调用大多数应该在模型中进行调用，这是我们推荐的最佳方式，有非常有利于代码风格的一致性。

// pageA/store.js

export default (app) => {
  const { service } = app;

  return {
    // 数据对象
    data: {},

    async getUserInfo() {
      // 调用服务接口
      this.data = await service.getUserInfo()
    }
  }
}

# 在视图中请求数据

最后，我们在视图中可以通过 useApp 访问到 store 中定义的方法体，然后发送请求。

// pageA/index.jsx
export default (app) => {
  const { app } = useApp()

  React.useEffect(() => {
    // 通过 pageA.store 访问
    await app.pageA.store.getUserInfo();
  }, [])
}
